# 会员系统开发进度（2026-03）

## 1. 本轮目标

在已有“视频下载 + AI 总结”能力基础上，落地最小可用商业化闭环：

- 用户账号体系（邮箱 + 密码）
- Stripe 会员支付（CNY）
- AI 权限控制（免费 5 次/日，VIP 不限）
- 密码找回（邮箱重置）
- 账单/订单历史查询

## 2. 已完成进度

### 阶段 A：后端基础能力

- 引入 SQLite 持久化（用户、订单、权益、Webhook 事件、AI 每日额度）
- 补充配置体系（JWT、Stripe、SMTP、额度参数）
- 启动时自动建表/迁移（兼容增量字段）

### 阶段 B：认证与支付核心链路

- 认证接口：注册、登录、`/me`
- 安全策略：密码 bcrypt 哈希、JWT 鉴权
- 支付接口：Checkout Session、Webhook、会员状态、订单列表
- 幂等策略：
  - 创建支付：业务 idempotency key + Stripe idempotency key
  - Webhook：`event_id` 去重
- 支付安全：Webhook 签名验签

### 阶段 C：前端接入与体验优化

- 登录/注册弹窗、会话持久化（token）
- 顶部会员状态展示 + 开通 VIP 入口
- AI 超限提示与升级引导（避免技术错误文案直出）
- 账单历史弹窗页（订单状态、金额、时间）

### 阶段 D：扩展功能

- 密码找回（请求重置邮件 + token 重置密码）
- 开发模式离线支付闭环（`/api/billing/mock/webhook-paid`）

## 3. 需求变更记录

- 原计划包含“邮箱验证激活”。
- 经确认，本轮回退邮箱验证激活能力（不再阻塞登录、不再保留验证接口）。
- 当前保留：注册/登录 + 密码找回 + 账单历史 + 会员支付。

## 4. 当前可验收能力

- 注册后可直接登录
- 免费用户 AI 总结超过 5 次会被拦截
- 点击“开通 VIP”可走 Stripe 支付链路
- 支付成功后会员状态更新
- 可查看订单历史
- 可通过邮箱完成密码重置

## 5. 已知限制与后续建议

- 下载任务状态仍为内存态，重启后丢失
- SMTP 通道不可用时，密码找回链路不可用
- 缺少后台运营能力（手工补单、退款、争议单流程）
- 建议下阶段补齐：
  - 退款/拒付处理
  - 管理后台（订单检索、用户检索、手动发放）
  - 审计日志与告警
