# 万能视频下载站 MVP - 项目总结文档

## 1. 项目阶段结论

当前已完成 MVP 的核心闭环能力：

- 链接解析（标题、时长、封面、格式列表）
- 单条/批量下载任务
- 下载进度与任务状态展示
- 本地落盘与打开文件位置
- B 站与 YouTube 关键兼容性问题修复
- AI 视频总结（SSE 流式、DeepSeek、平台字幕提取）

项目达到“可演示、可本地使用、可继续扩展”的阶段目标。

## 2. 已落地能力清单

## 2.1 后端（FastAPI + yt-dlp）

- `POST /api/video/inspect`：解析视频信息
- `POST /api/video/download`：创建单任务下载
- `POST /api/video/download/batch`：创建批量任务
- `GET /api/video/tasks`、`GET /api/video/tasks/{task_id}`：任务查询
- `GET /api/video/thumbnail`：缩略图代理，提升封面稳定性
- `GET /api/video/config`：返回运行时下载目录
- `POST /api/video/open-path`：一键打开本地路径

## 2.2 前端（Vue3）

- 单条/批量输入模式切换
- 格式下拉与分离流标签（仅视频/仅音频/音视频）
- 分离流提示文案（自动合并）
- 任务进度条、状态标签、错误提示
- 下载目录展示与“打开文件位置”按钮
- AI 总结区（SSE 流式状态、增量内容、结构化结果渲染）

## 2.3 AI 总结能力（新增）

- `POST /api/ai-summary/stream`：SSE 流式返回总结过程与结果
- 总结模型：DeepSeek 官方 API，默认 `deepseek-chat`
- 输出结构：`summary`、`outline[]`、`key_points[]`
- 字幕来源：
  - 主路径：`yt-dlp` 字幕字段（`subtitles` / `automatic_captions`）
  - 兜底路径：B 站 `player/wbi/v2` 字幕接口提取
- 运行配置：
  - `DEEPSEEK_API_KEY`、`DEEPSEEK_BASE_URL`、`DEEPSEEK_MODEL`
  - `DEEPSEEK_TIMEOUT_SECONDS`、`DEEPSEEK_PROXY_URL`
  - `YTDLP_COOKIEFILE`、`YTDLP_COOKIES_FROM_BROWSER`

## 3. 关键问题与修复记录

## 3.1 封面不显示

问题原因：

- 部分平台封面地址为 `http`、防盗链或跨域触发前端加载失败。

修复方案：

- 后端规范化缩略图 URL（优先 https）
- 前端统一走后端代理接口加载封面

## 3.2 B 站下载失败/无声

问题原因：

- 平台经常返回 DASH 分离流（视频流与音频流分开）
- 用户手动选“仅视频”时会出现无声
- 缺少 ffmpeg 时无法合并

修复方案：

- 自动格式策略：若选视频流，自动补 `bestaudio` 合并
- 增加 ffmpeg 本地能力并在后端自动检测
- 强制覆盖重下，避免旧无声文件被“已下载”逻辑复用

## 3.3 YouTube 403

问题原因：

- 默认客户端在当前网络环境被风控

修复方案：

- YouTube 链接启用 `android_vr` 提取客户端策略

## 3.4 AI 总结可用性问题（新增）

问题原因：

- 部分 B 站视频在 `yt-dlp` 中字幕字段为空，但页面实际存在字幕
- 本机代理/证书环境可能导致 DeepSeek TLS 握手异常

修复方案：

- 新增 B 站字幕 API 兜底提取（`x/web-interface/view` + `x/player/wbi/v2`）
- 增加显式代理配置 `DEEPSEEK_PROXY_URL`，并默认不信任系统代理环境变量
- 前端 SSE 完成态修正：结束后状态正确显示 `done/failed`

## 4. 当前架构状态

- 前端：`frontend/src/App.vue`
- 后端入口：`api/main.py`
- 路由：`api/routers/video.py`
- 路由：`api/routers/ai_summary.py`
- 下载服务：`api/services/downloader.py`
- 字幕服务：`api/services/subtitle_extractor.py`
- AI 总结服务：`api/services/ai_summary.py`
- 任务管理：`api/services/tasks.py`
- 文档：`docs/需求分析文档.md`、`docs/方案设计文档.md`

## 5. 当前已知约束

- 任务状态仅内存存储，服务重启后丢失
- 暂无账号体系与权限控制
- 暂无支付与配额逻辑
- 暂未接入持久化日志与监控
- AI 总结仍依赖平台可用字幕，部分受限视频可能需要登录态/cookie

## 6. 下一阶段建议（按优先级）

1. 引入数据库（用户、任务、订阅）
2. 引入异步任务队列（Redis + Worker）
3. 接入 Stripe 会员体系（并发与配额权益）
4. 增加 AI 能力（视频总结、字幕翻译）
5. 生产部署与监控（日志、告警、审计）
