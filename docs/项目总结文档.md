# 万能视频下载站 MVP - 项目总结文档

## 1. 项目阶段结论

当前已完成 MVP 的核心闭环能力：

- 链接解析（标题、时长、封面、格式列表）
- 单条/批量下载任务
- 下载进度与任务状态展示
- 本地落盘与打开文件位置
- B 站与 YouTube 关键兼容性问题修复

项目达到“可演示、可本地使用、可继续扩展”的阶段目标。

## 2. 已落地能力清单

## 2.1 后端（FastAPI + yt-dlp）

- `POST /api/video/inspect`：解析视频信息
- `POST /api/video/download`：创建单任务下载
- `POST /api/video/download/batch`：创建批量任务
- `GET /api/video/tasks`、`GET /api/video/tasks/{task_id}`：任务查询
- `GET /api/video/thumbnail`：缩略图代理，提升封面稳定性
- `GET /api/video/config`：返回运行时下载目录
- `POST /api/video/open-path`：一键打开本地路径

## 2.2 前端（Vue3）

- 单条/批量输入模式切换
- 格式下拉与分离流标签（仅视频/仅音频/音视频）
- 分离流提示文案（自动合并）
- 任务进度条、状态标签、错误提示
- 下载目录展示与“打开文件位置”按钮

## 3. 关键问题与修复记录

## 3.1 封面不显示

问题原因：

- 部分平台封面地址为 `http`、防盗链或跨域触发前端加载失败。

修复方案：

- 后端规范化缩略图 URL（优先 https）
- 前端统一走后端代理接口加载封面

## 3.2 B 站下载失败/无声

问题原因：

- 平台经常返回 DASH 分离流（视频流与音频流分开）
- 用户手动选“仅视频”时会出现无声
- 缺少 ffmpeg 时无法合并

修复方案：

- 自动格式策略：若选视频流，自动补 `bestaudio` 合并
- 增加 ffmpeg 本地能力并在后端自动检测
- 强制覆盖重下，避免旧无声文件被“已下载”逻辑复用

## 3.3 YouTube 403

问题原因：

- 默认客户端在当前网络环境被风控

修复方案：

- YouTube 链接启用 `android_vr` 提取客户端策略

## 4. 当前架构状态

- 前端：`backend/src/App.vue`
- 后端入口：`api/main.py`
- 路由：`api/routers/video.py`
- 下载服务：`api/services/downloader.py`
- 任务管理：`api/services/tasks.py`
- 文档：`docs/需求分析文档.md`、`docs/方案设计文档.md`

## 5. 当前已知约束

- 任务状态仅内存存储，服务重启后丢失
- 暂无账号体系与权限控制
- 暂无支付与配额逻辑
- 暂未接入持久化日志与监控

## 6. 下一阶段建议（按优先级）

1. 引入数据库（用户、任务、订阅）
2. 引入异步任务队列（Redis + Worker）
3. 接入 Stripe 会员体系（并发与配额权益）
4. 增加 AI 能力（视频总结、字幕翻译）
5. 生产部署与监控（日志、告警、审计）
